2 - Instalar o Java

  1. Verifique o pacote Java esta instalado;

[root@localhost ~]$  yum search java | grep -i --color JDK

  1. Instale o Java;

[root@localhost ~]$  yum install java-1.7.0-openjdk java-1.7.0-openjdk-devel

  1. Set the JAVA_HOME environment variable on a CentOS Linux

[root@localhost ~]$ ls -l /usr/lib/jvm/

[root@localhost ~]$  export JAVA_HOME=/usr/lib/jvm/jre-1.7.0-openjdk.x86_64/

  1. Crie um programa jva para testar;

[root@localhost ~]$  gedit HelloWorld.java

public class HelloWorld {

    public static void main(String[] args) {

        System.out.println("Hello, World!  Este é apenas um teste, by LabFiap!");

    }

}

  1. Compile o programa;

[root@localhost ~]$  javac HelloWorld.java

  1. Teste o programa;

[root@localhost ~]$  java HelloWorld

3 - Criar Usuário

  1. Crie o usuário hadoop em todos os nodes;

[root@localhost ~]$  useradd hadoop

  1. Altere a senha do usuário hadoop para “labfiap”;

[root@localhost ~]$  passwd hadoop

  1. Altere sua sessão no terminal para o usuário hadoop ou faça o login na VM com o usuário hadoop;

[root@localhost ~]$  su - hadoop

  1. Adicione as variáveis ambientes no perfil no usuário do usuário, repita este passo em cada nó;

[hadoop@localhost ~]$  gedit ~/.bashrc

# Hadoop variables

export HADOOP_INSTALL="/hadoop-2.9.0"

export PATH=$PATH:$HADOOP_INSTALL/bin

export PATH=$PATH:$HADOOP_INSTALL/sbin

export HADOOP_MAPRED_HOME=$HADOOP_INSTALL

export HADOOP_COMMON_HOME=$HADOOP_INSTALL

export HADOOP_HDFS_HOME=$HADOOP_INSTALL

export YARN_HOME=$HADOOP_INSTALL

export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_INSTALL/lib/native

export HADOOP_OPTS="-Djava.library.path=$HADOOP_INSTALL/lib/native"

  1. Carregando as configurações no profile do usuário da sessão;

[hadoop@localhost ~]$ . ~/.bashrc

[hadoop@localhost ~]$ sudo  gedit /etc/hosts

192.168.188.10  hadoop-master

5 - Configurando SSH com autenticação de chave RSA

  1. Configurando SSH com autenticação de chave RSA, repita este passo para cada nó do cluster;

[hadoop@localhost ~]$ ssh-keygen -t rsa

Obs: tecle enter para confirmar cada passo até o final;

Copie a chave para as outras máquinas do cluster:

[hadoop@localhost ~]$ ssh-copy-id -i ~/.ssh/id_rsa.pub hadoop@hadoop-master

[hadoop@localhost ~]$ ssh-copy-id -i ~/.ssh/id_rsa.pub hadoop@hadoop-slave1

[hadoop@localhost ~]$ ssh-copy-id -i ~/.ssh/id_rsa.pub hadoop@hadoop-slave2

Altere as permissões em todos os nós do cluster conforme abaixo:

[hadoop@localhost ~]$ chmod 0600 ~/.ssh/authorized_keys

6 - Configurando o Path do Hadoop

Os pacotes de instalação estão no diretório /install;

  1. Descompactando os pacotes de instalação;

[hadoop@localhost ~]$  tar -xzvf /install/hadoop-2.9.0.tar.gz -C /hadoop-2.9.0 --strip-components=1

7 - Configurando o Hadoop

Os arquivos de configuração do Hadoop estão no diretório /hadoop/etc/hadoop

  1. Edite o arquivo core-site.xml ;

[hadoop@localhost ~]$  gedit /hadoop-2.9.0/etc/hadoop/core-site.xml

<configuration>

<property>

        <name>fs.defaultFS</name>

        <value>hdfs://hadoop-master:9000/</value>

</property>

<property>

        <name>dfs.permissions</name>

        <value>false</value>

</property>

<property>

        <name>hadoop.tmp.dir</name>

        <value>/hadoop-2.9.0/tmp</value>

</property>

</configuration>

  1. Edite o arquivo hdfs-site.xml;

[hadoop@localhost ~]$  gedit /hadoop-2.9.0/etc/hadoop/hdfs-site.xml

<configuration>

<property>

 <name>dfs.secondary.http.address</name>

        <value>hadoop-master:50090</value>

        <description>Secondary NameNode hostname</description>

</property>

<property>

        <name>dfs.data.dir</name>

        <value>/hadoop-2.9.0/hdfs/datanode</value>

        <final>true</final>

</property>

<property>

        <name>dfs.name.dir</name>

        <value>/hadoop-2.9.0/hdfs/namenode</value>

        <final>true</final>

</property>

<property>

        <name>dfs.blocksize</name>

        <value>67108864</value>

</property>

<property>

        <name>dfs.replication</name>

        <value>1</value>

</property>

</configuration>

  1. Edite o arquivo mapreduce-site.xml;

[hadoop@localhost ~]$  gedit /hadoop-2.9.0/etc/hadoop/mapred-site.xml

<configuration>

<property>

        <name>mapreduce.framework.name</name>

        <value>yarn</value>

</property>

</configuration>

  1. Edite o arquivo yarn-site.xml;

[hadoop@localhost ~]$  gedit /hadoop-2.9.0/etc/hadoop/yarn-site.xml

<configuration>

<!-- Site specific YARN configuration properties -->

<property>

        <name>yarn.nodemanager.aux-services</name>

        <value>mapreduce_shuffle</value>

</property>

<property>

        <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>

        <value>org.apache.hadoop.mapred.ShuffleHandler</value>

</property>

<property>

        <name>yarn.scheduler.minimum-allocation-mb</name>

        <value>256</value>

</property>

<property>

        <name>yarn.scheduler.maximum-allocation-mb</name>

        <value>2048</value>

</property>

<property>

        <name>yarn.scheduler.maximum-allocation-vcores</name>

        <value>1</value>

</property>

<property>

        <name>yarn.resourcemanager.resource-tracker.address</name>

        <value>hadoop-master:8025</value>

</property>

<property>

        <name>yarn.resourcemanager.scheduler.address</name>

        <value>hadoop-master:8030</value>

</property>

<property>

        <name>yarn.resourcemanager.address</name>

        <value>hadoop-master:8040</value>

</property>

</configuration>

  1. Altere as variáveis ambientes conforme abaixo no final do arquivo;

[hadoop@localhost ~]$  gedit /hadoop-2.9.0/etc/hadoop/hadoop-env.sh

export JAVA_HOME="/usr/lib/jvm/jre-1.7.0-openjdk"

export HADOOP_OPTS="$HADOOP_OPTS -Djava.net.preferIPv4Stack=true"

export HADOOP_CONF_DIR="/hadoop-2.9.0/etc/hadoop"

  1. Alterar os arquivos Master e Slaves para definição do cluster;

[hadoop@localhost ~]$  gedit /hadoop-2.9.0/etc/hadoop/masters

hadoop-master

[hadoop@localhost ~]$  gedit /hadoop-2.9.0/etc/hadoop/slaves

hadoop-master

hadoop-slave1

  1. Copiando as configurações para os outros nós a partir do hadoop-master;

[hadoop@localhost ~]$  sudo scp -r /hadoop-2.9.0 hadoop-slave1:/

[hadoop@localhost ~]$  ssh hadoop-slave1

  1. Formatar o cluster  HDFS;

[hadoop@localhost ~]$  hdfs namenode -format

Master & Slave

Essa localização é dada pelo endereço de rede ou pelo apelido desses recursos os respectivos arquivos masters e slaves. Sabemos que no modo pseudo-distribuído estamos simulando uma execução distribuída, dessa forma, para esse modo, esses locais serão sempre os mesmos.
