Exercício 1: Ingestão e Consultas Relacionais

A DataPro está tentando responder a seguinte questão de negócio:  

Que produtos nossos clientes gostam de comprar?

Para responder a esta pergunta, o primeiro passo pode ser olhar para os dados da transação, que deve indicar o que os clientes realmente compram e gostariam de comprar, certo?

Este é provavelmente algo que você pode fazer em seu ambiente RDBMS tradicional, mas um benefício do Apache Hadoop é que você pode fazê-lo em maior escala e menor custo, no mesmo sistema que você também pode usar para muitos outros tipos de análise.

O que este exercício demonstra é como fazer exatamente a mesma coisa que você já sabe como fazer, mas em Hadoop. A integração perfeita é importante ao avaliar qualquer nova infraestrutura. Por isso, é importante ser capaz de fazer o que você normalmente faz e não interromper quaisquer relatórios de BI ou cargas de trabalho regulares sobre o conjunto de dados que você planeja migrar.

Para analisar os dados de transação na nova plataforma, precisamos fazer a ingestão no Sistema de Arquivos Distribuídos Hadoop (HDFS). Precisamos encontrar uma ferramenta que transfira facilmente dados estruturados de um RDBMS para HDFS, preservando a estrutura. Isso nos permite consultar os dados, mas não interferir ou quebrar qualquer carga de trabalho regular sobre ele.

Apache Sqoop, que faz parte do Ecossistema, é essa ferramenta. Através do Sqoop podemos carregar automaticamente nossos dados relacionais de MySQL em HDFS, preservando a estrutura. Com alguns parâmetros de configuração adicionais, podemos dar um passo adiante e carregar esses dados relacionais diretamente para um formulário pronto para ser consultado pelo Apache Impala (incubando), o banco de dados analítico MPP incluso no CDH.

Você deve primeiro fazer logon no nó mestre do cluster por meio de um terminal. Em seguida, inicie o job Sqoop:

sqoop import-all-tables \
   --m 1 \
   --connect jdbc:mysql://127.0.0.1:3306/retail_db \
   --username=retail_dba \
   --password=cloudera \
   --compression-codec=snappy \
   --as-parquetfile \
   --warehouse-dir=/user/hive/warehouse \
   --hive-import

Esse comando pode demorar um pouco para ser concluído, devido a alta carga. Ele está lançando jobs MapReduce para puxar os dados de nosso banco de dados MySQL e gravar os dados para HDFS em paralelo, distribuídos através do cluster em formato Apache Parquet . Também está criando tabelas para representar os arquivos HDFS no Impala/Apache Hive com o esquema correspondente.

Parquet é um formato projetado para aplicações analíticas no Hadoop. Em vez de agrupar seus dados em linhas como formatos de dados típicos, agrupa seus dados em colunas. Isso é ideal para muitas consultas analíticas em que, em vez de recuperar dados de registros específicos, você está analisando relacionamentos entre variáveis ​​específicas em muitos registros. Parquet é projetado para otimizar o armazenamento e recuperação de dados nesses cenários.

Nota: O número de arquivos .parquet mostrados será igual ao que foi passado para o Sqoop com o parâmetro -m. Esse é o número de 'mappers' que o Sqoop usará em seus trabalhos MapReduce. Poderá também ser pensado como o número de ligações simultâneas ao seu banco de dados, ou o número de discos/nós de dados que você deseja espalhar os dados. Assim, em um único nó você verá apenas um, mas clusters maiores terão um maior número de arquivos.

O Hive e o Impala também permitem que você crie tabelas definindo um esquema sobre arquivos existentes com instruções 'CREATE EXTERNAL TABLE', semelhantes aos bancos de dados relacionais tradicionais. Mas o Sqoop já criou essas tabelas para nós, para que possamos prosseguir e consultá-las.

Vamos usar o aplicativo Impala do Hue para consultar nossas tabelas. Hue fornece uma interface baseada na web para muitas das ferramentas em CDH e pode ser encontrado na porta 8888 de seu nó de Gerenciamento. Na VM QuickStart, o nome de usuário do administrador para Hue é 'cloudera' e a senha é 'cloudera'.

Quando estiver dentro do Hue, clique em Query Editors e abra o Impala Query Editor.

Para poupar tempo durante as consultas, o Impala não pesquisa constantemente alterações de metadados. Portanto, a primeira coisa que devemos fazer é dizer ao Impala que seus metadados estão desatualizados. Então podemos ver nossas tabelas, prontas para serem consultadas:

invalidate metadata;

show tables;

Você também pode clicar no ícone "Refresh Table List" na esquerda para ver suas novas tabelas no menu lateral.

Agora que seus dados de transações estão prontamente disponíveis para consultas estruturadas no Hadoop, é hora de responder à pergunta de negócios da DataPro. Copie e cole ou digite as seguintes consultas de exemplo SQL padrão para calcular a receita total por produto e mostrando os 10 principais produtos geradores de receita:

-- Mais populares categorias de produtos

select c.category_name, count(order_item_quantity) as count
from order_items oi
inner join products p on oi.order_item_product_id = p.product_id
inner join categories c on c.category_id = p.product_category_id
group by c.category_name
order by count desc
limit 10;

Você deve ver os resultados do seguinte formulário:

Limpe a consulta anterior e substitua-a pelo seguinte:

-- os 10 principais produtos geradores de receitas

select p.product_id, p.product_name, r.revenue
from products p inner join
(select oi.order_item_product_id, sum(cast(oi.order_item_subtotal as float)) as revenue
from order_items oi inner join orders o
on oi.order_item_order_id = o.order_id
where o.order_status <> 'CANCELED'
and o.order_status <> 'SUSPECTED_FRAUD'
group by order_item_product_id) r
on p.product_id = r.order_item_product_id
order by r.revenue desc
limit 10;

Você pode perceber que nós dissemos ao Sqoop para importar os dados para o Hive mas usamos o Impala para consultar os dados. Isso ocorre porque o Hive e o Impala podem compartilhar arquivos de dados e os metadados da tabela. O Hive trabalha compilando consultas SQL em jobs MapReduce, o que o torna muito flexível, enquanto o Impala executa consultas e é construído a partir do zero para ser o mais rápido possível, o que o torna melhor para a análise interativa.

Conclusão:

Você aprendeu como consultar tabelas colunares usando Impala e que você pode usar interfaces e ferramentas regulares (like SQL) dentro de um ambiente Hadoop. A idéia aqui é que você pode fazer os mesmos relatórios que você costuma fazer, mas a arquitetura de Hadoop vs sistemas tradicionais fornece muito mais escala e flexibilidade.
