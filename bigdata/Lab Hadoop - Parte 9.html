O Impala composto de 3 componentes principais:

1.        Impalad: Processo que fica ativo em cada um dos DataNodes para processar as queries, ler e gravar arquivos no HDFS/HBase.

Query Planner: Frontend desenvolvido em Java para realizar o parse, analisar e planejar as queries SQL.

Query Coordinator / Executor: Backend desenvolvido em C++ para coordenar a comunicao e execuo dos fragmentos no plano. O Thrift   utilizado para realizar a RPC e comunicao com outros Impalads do cluster.

2.        Catalog Service: Armazena os metadados e servio de catlogo. Fica ativo em um servidor por cluster, geralmente fica no mesmo servidor do Hive metastore - EdgeNode.

Metadata: bancos de dados, tabelas, views, colunas, etc. Tambm armazena a localizao dos arquivos e dos blocos.

Catalog: Envia o metadados para todos os impalads via statestore. a interface persistente do de armazenamento de metadados entre o metastore do Hive e o Impala.

3.        Statestore: Servio centralizado de estado do repositrio para gerenciar os status dos Impalads, name service e metadados. Realiza os heartbeats nos Impalads.

Fica ativo em um servidor por cluster, geralmente fica no mesmo servidor do NameNode ou no do Catalog.

Exemplo Benchmark Impala e Hive

  1. Acessar o Impala (Obs: Avaliar se o servio esta ativo no Cloudera Manager);

[cloudera@localhost ~]$  impala-shell

  1. Criar uma nova tabela;

impala> drop table bolsafamilia2;

create table BolsaFamilia2 (

estado string,

codigo_municipio int,

nome_municipio string,

codigo_funcao int,

codigo_subfuncao int,

codigo_programa int,

codigo_acao int,

nis_favorecido string,

nome_favorecido string,

fonte_finalidade string,

valor_parcela float)

COMMENT 'Informacao Bolsa Familia via Impala'

row format delimited fields terminated by '\t' stored as textfile;

  1. Efetuar a carga de dados na tabela criada;

impala> load data inpath "/user/cloudera/201411_BolsaFamiliaFolhaPagamento.csv" into table bolsafamilia2;

  1. Executar um select (*obs: Note a diferena no tempo de resposta em relao ao Hive);

impala> select estado, sum(valor_parcela) from bolsafamilia2 group by estado;

  1. Criar uma nova tabela usando PARQUET file format;

Parquet um formato de arquivo binrio orientada a coluna, altamente eficiente para consultas massivas;

impala> create table bolsafamilia3 LIKE bolsafamilia2 STORED AS PARQUET;

  1. Popular a nova tabela PARQUET;

impala> insert overwrite table bolsafamilia3 select * from bolsafamilia2 where valor_parcela < 1000.00;

  1. Consultar nova tabela PARQUET; (comparar performance com a tabela bolsafamilia2)

impala> select estado, sum(valor_parcela) from bolsafamilia3 group by estado;

  1. Consulta com filtro;

impala> select * from bolsafamilia3 where nis_favorecido='00010010059498';
